Blue Book Phenomenology

What 10,614 declassified case files reveal when the archive speaks for itself

— case files • 1947–1969 • 5 phenomenological clusters • ~130,000 pages OCR'd

Project Blue Book was the U.S. Air Force's official investigation of unidentified flying objects, running from 1947 to 1969. The complete declassified archive — 10,614 case files across approximately 130,000 pages — has been OCR'd, cleaned, and embedded in high-dimensional vector space. Rather than imposing the Air Force's own categories, this analysis lets the phenomenological structure of witness testimonies emerge from the data itself.

1. Five Phenomenological Clusters

When 17 structured features — shape, color, sound, movement pattern, physical effects, witness type, and more — are extracted from each case file and clustered, five distinct phenomenological types emerge. These categories were invisible to the Air Force's own classification system and to raw text embeddings.

Feature-Based Embedding — The Five Clusters
Each point is one case file. Proximity = phenomenological similarity. Click any point to read the witness narrative, extracted features, and original scanned document.
Method: 17 features encoded as 82-column binary matrix → UMAP to 2D → K-means (k=5). Embedding: nomic-embed-text-v1.5 (768d, 8192-token context).
Five clusters emerge that were invisible to the Air Force's classification and to raw text embeddings. Four share a common phenomenological vocabulary (lights, movement, brief duration) and differ by observer context and effect intensity. The fifth — Structured Craft — has a categorically different feature signature.

9. How We Got Here

The raw archive defeats naive computational analysis. Two problems had to be solved before the phenomenological clusters could emerge.

The Inscription Problem: Full-Document Embedding
What does the archive look like when you embed everything — forms, metadata, testimony?
Method: nomic-embed-text-v1.5 on complete OCR text → UMAP to 2D → HDBSCAN. 185 clusters, 33% noise.
The forms work. The AF Form 112 dominates the semantic signal. Clusters correspond to states and decades, not to what witnesses experienced. The institutional frame must be computationally separated from the witness voice.
The Mega-Cluster: Narrative Embedding
What happens when you embed only the extracted witness testimony?
Method: Claude Haiku extracts witness narrative from each case → nomic-embed-text-v1.5 → UMAP to 2D.
The mega-cluster problem. Witness language is more uniform than the experiences it describes. Semantic similarity collapses phenomenologically distinct events. The solution: cluster on structured features, not text.

10. Outstanding Work

The 701 Overlay (in progress): Cross-reference with the Brad Sparks catalog to tag the 701 officially unidentified cases. The current approximation uses LLM-extracted conclusions; the canonical list would confirm whether the unidentified cases cluster differently in feature space.

VASCO Transient Cross-Reference: Correlate Blue Book case dates with the April 1950 Palomar Sky Survey photographic plate transients (Villarroel et al.) and nuclear test dates. Three-legged temporal correlation: witness sightings × photographic anomalies × nuclear activity.

Nuclear Testing Correlation: Villarroel and Bruehl (2025, Scientific Reports) found transients 45% more likely within one day of nuclear tests. Blue Book case dates are indexed — the temporal correlation is directly testable.

Semantic Search: Deploy a Supabase Edge Function to embed query text and search by narrative similarity, not just keyword match.

PDF Archive: Upload all ~10,800 original scanned PDFs to Supabase Storage for self-contained access (1940s decade complete, 1950s/1960s in progress).

Collaborative Annotations: OTP authentication for research collaborators. Shared annotation system for tagging cases with observations, questions, and cross-references.
